## 👥 참석 전문가 그룹 & 아젠다

### **Group 1: 게임 디자인/플레이 경험 (Game Design & Engagement)**
**참석자**: GD Lead, 밸런스 디자이너, 컨텐츠 기획자

#### 주요 논의 사항
1. **데크 빌딩 게임의 진행성 도입**
    - 현재: 순수 주식 거래 시뮬레이터
    - 개선안: 카드(종목) 컬렉션 요소 추가
    - "직원 스킬 카드" + "투자 전략 카드" 시스템

2. **브루마블식 운 + 전략의 밸런스**
    - 턴 기반 액션(매월 투자 결정)에 랜덤 요소 추진
    - "시장 뉴스 카드" 드로우 → 기회/위협 발생
    - 플레이어 선택지의 다양화 (보수/공격/도박)

3. **도트 게임의 쾌감 피드백**
    - 체감 난이도 곡선(쉬운→중간→어려움)
    - 성공/실패 시 즉각적인 시각/음향 피드백
    - 마일스톤 시스템 (1개월 → 1년 → 5년 달성감)

#### 📌 즉시 실행 아이디어
```
✅ "Milestone Craving System"
   - 자동으로 다음 목표 표시 (현재 자산 대비 +50% 등)
   - 달성 시 "🎉 ACHIEVEMENT UNLOCKED" 같은 표현

✅ "Monthly Card Draw" 시스템
   - 매월 3개 뉴스 카드 중 2개 선택 (1개는 필수)
   - "기술 주식 대폭락" vs "에너지 급등" 중 선택
   - 선택지가 없는 "랜덤 폭탄 이벤트"도 10% 확률

✅ "직원 카드화"
   - Rarity: 일반(초록)→레어(파랑)→에픽(보라)→레전드(금색)
   - "스킬 카드" 추가 선택지 (매월 1명이 1개 스킬 습득 가능)
```

### **Group 2: UI/UX & Visual Design**
**참석자**: UX/UI 디자이너, 인터랙션 디자이너, 시각 디자인 리드

#### 주요 논의 사항
1. **Windows 95 테마의 현대적 진화**
    - 현재: 진정성 있는 레트로 (고착)
    - 개선안: "Retro-Modern Fusion" (톤은 95, 애니메이션은 2024)
    - 예: Glassmorphism + 픽셀 아트 하이브리드

2. **정보 과부하 해결 (Information Architecture)**
    - 현재 문제: 차트, 뉴스, 직원, 거래 창이 흩어짐
    - 개선안: **대시보드 통합** (메인 화면 한눈에 현황)
        - Left Sidebar: 빠른 액션 (매수/매도)
        - Center: 포트폴리오 헬스 게이지 + 다음 목표
        - Right Sidebar: 직원 카드 + 뉴스 피드

3. **터치/마우스 상호작용 개선**
    - 드래그 앤드 드롭 강화 (직원 배치, 가구 배치)
    - 휠 스크롤 감도 조정 (차트 줌인/아웃)
    - 우클릭 컨텍스트 메뉴 (빠른 매수, 정보 조회)

#### 📌 즉시 실행 아이디어
```
✅ "Main Dashboard 리뉴얼"
   화면 분할: 
   ┌─────────────────┬─────────────────┐
   │ 거래 패널       │ 포트폴리오      │
   │ (빠른 매수/매도) │ (원형 차트)     │
   ├─────────────────┼─────────────────┤
   │ 직원 대열       │ 뉴스 피드       │
   │ (카드 뷰)       │ (스크롤)        │
   └─────────────────┴─────────────────┘

✅ "Hover States & Tooltips"
   - 모든 주식에 마우스 호버 시 미리보기
   - 직원 호버 시 현재 스킬/스트레스 상태 표시

✅ "Dark/Light Mode Toggle"
   - 95년식 그레이 테마
   - 신형 다크 테마 (고대비, 블루라이트 컷)
   - 사용자 선택 저장
```

### **Group 3: 프로그래매틱 및 기술 (Programming & Tech)**
**참석자**: 소프트웨어 아키텍트, 성능 엔지니어, 게임 시스템 프로그래머

#### 주요 논의 사항
1. **애니메이션 시스템 강화**
    - 현재: 정적인 UI 업데이트
    - 개선안: Motion.js 극대화
        - 카드 플립 애니메이션 (직원, 뉴스)
        - 숫자 카운팅 애니메이션 (자산 변화)
        - 입자 효과 (돈 날기, 수익/손실)

2. **상태 머신 개선 (State Management)**
    - 현재: 단순 store 업데이트
    - 개선안: FSM으로 "거래 흐름" 관리
        - "주식 선택" → "수량 입력" → "확인" → "체결"
        - 각 단계에서 취소 가능, 명확한 피드백

3. **사운드 시스템 확장**
    - 현재: 8비트 OSC 효과음
    - 개선안: 신발라드 컨셉
        - "구매 성공": 동전 소리
        - "손실": 슬픈 8비트
        - "첫 백만원": 팡파레 (거품 터지는 소리)

#### 📌 즉시 실행 아이디어
```
✅ "Transactional Animation Sequence"
   주식 매수 클릭 → 
   1. 카드 플립 (주식명 노출)
   2. 가격 카운팅 (0 → 구매금액)
   3. 숫자 상승 (포트폴리오 자산 ↑)
   4. 파티클 (돈 날기)
   5. 사운드 (동전 소리 + 신호음)
   총 1.5초

✅ "Progress Indicator System"
   - 30년 진행도 (상단 바)
   - 월간 목표 진행도 (좌측)
   - 직원 레벨 진행도 (오른쪽)
   - 모두 색상 코딩 (초록=좋음, 노랑=주의, 빨강=위험)

✅ "Notification Center"
   - 중요도별 알림 필터 (수익/손실/뉴스/직원)
   - 불린 시스템 대신 "뱃지" 표시
   - 오디오 알림 온/오프 토글
```

### **Group 5: 게임 밸런스 & 경제학 (Game Balance & Economics)**
**참석자**: 경제 시뮬레이션 전문가, 밸런스 디자이너, QA 리드

#### 주요 논의 사항
1. **스킬 트리 개선**
    - 현재: 선형적 레벨업
    - 개선안: "Branching Skill Tree"
        - 기본 경로: 트레이더 → 펀드 매니저 → CEO
        - 대체 경로: 애널리스트 → 리서처 → 컨설턴트
        - 각 경로별 보너스 (트레이더: +거래수익, 리서처: +이벤트 예측)

2. **경제 순환 설계 (Economy Loop)**
    - 문제: 일단 부자 되면 계속 부자 (빈익빈부익부)
    - 개선안: "Economic Pressure"
        - 자산 1억 초과 시 월 세금 발생
        - 과도한 주식 보유 시 "포트폴리오 리밸런싱" 압박
        - 진정한 스킬이 필요한 "고난이도 존"

3. **난이도 곡선 동적 조정 (Difficulty Scaling)**
    - 현재: 시작 시에만 난이도 선택
    - 개선안: "Adaptive Difficulty"
        - 플레이어 성적이 너무 좋으면 → 자동으로 이벤트 증가
        - 계속 손실 나면 → 잠깐 호재성 뉴스 제공
        - 항상 "도전감 있는 수준" 유지

#### 📌 즉시 실행 아이디어
```
✅ "Branching Skill Tree"
   직원 교육 시 선택지:
   ┌─────────────────┬─────────────────┐
   │ 트레이딩 교육   │ 분석 교육       │
   │ (매매 수익 +20%)│ (예측력 +30%)   │
   ├─────────────────┼─────────────────┤
   │ 펀드 운영       │ 컨설팅          │
   │ (자산 관리 +10%)│ (패턴 인식 +25%)│
   └─────────────────┴─────────────────┘

✅ "Tax & Regulatory System"
   자산 1억 초과 → 매월 세금 (자산의 0.5%)
   자산 10억 초과 → 포지션 크기 제한 (보유주 최대 20%)
   → 계속해서 신선한 전략 필요

✅ "Dynamic Difficulty Adjustment"
   IF 플레이어 수익률 > 50% (연율):
      → 다음 달 "보험사 파산 위기" 같은 초대형 이벤트
   IF 플레이어 손실률 > 20% (월간 연속):
      → "FED 기준금리 인하" 같은 구호금 제공
```
